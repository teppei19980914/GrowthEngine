# ユメログ 要件定義書

## 1. プロジェクト概要

| 項目 | 内容 |
|---|---|
| プロジェクト名 | ユメログ (YumeLog) |
| リポジトリ | GrowthEngine |
| 対象プラットフォーム | Web（GitHub Pages）、Windows、Android、iOS |
| 開発言語 | Dart / Flutter |
| 現行バージョン | 1.0.0 |

## 2. 背景と目的

### 背景

- 夢や目標を持っていても、日常に埋もれて行動に移せない人が多い
- 夢を言語化し、具体的なステップに分解し、日々の行動として実行する仕組みが必要
- 既存のタスク管理ツールは「効率化」に焦点を当てており、「夢の実現」という視点が欠けている

### 目的

- ユーザーが夢を言語化し、目標→タスクに分解して実行できるアプリを提供する
- 活動の継続をゲーミフィケーション（星座システム）で支援する
- Web体験版で気軽に始められ、ネイティブ版で本格運用できる環境を提供する

## 3. ステークホルダー

| 役割 | 対象 |
|---|---|
| 開発者 | 須山哲平 |
| 対象ユーザー | 夢や目標を持ちたい/実現したい全ての人 |
| 運用環境 | GitHub Pages（Web版）、各OS（ネイティブ版） |

## 4. 機能要件

### 4.1 コア機能

| ID | 機能 | 説明 | 優先度 |
|---|---|---|---|
| F-001 | 夢の管理 | 夢の登録・編集・削除。カテゴリ分類（8種） | 必須 |
| F-002 | 目標の管理 | 夢に紐づく目標の登録・編集・削除。What/When/How フレームワーク | 必須 |
| F-003 | タスクの管理 | 目標に紐づくタスクの登録・編集・削除。開始日/終了日/進捗率 | 必須 |
| F-004 | スケジュール表示 | タスクをタイムライン形式で可視化。横スクロール・日付ジャンプ対応 | 必須 |
| F-005 | 活動ログ記録 | タスクごとの活動時間を記録 | 必須 |
| F-006 | 書籍管理 | 書籍の登録・ステータス管理・読書スケジュール・読了レビュー | 必須 |
| F-007 | 統計表示 | 活動時間・日数・連続記録・目標別統計・書籍統計 | 必須 |
| F-008 | ダッシュボード | カスタマイズ可能なウィジェットグリッドで活動状況を一覧表示 | 必須 |

### 4.2 エンゲージメント機能

| ID | 機能 | 説明 | 優先度 |
|---|---|---|---|
| F-009 | 星座システム | 活動時間に応じて星が灯り、12星座が完成するゲーミフィケーション | 必須 |
| F-010 | 実績通知 | 累計時間・活動日数・連続記録のマイルストーン達成通知 | 必須 |
| F-011 | 受信ボックス | リマインダー・実績・開発者通知を一元表示 | 必須 |
| F-012 | 期限リマインダー | タスク/目標の期限7日前・当日・超過を自動通知 | 必須 |
| F-013 | チュートリアル | 初回アクセス時のインタラクティブガイド | 必須 |

## 5. 非機能要件

### 5.1 パフォーマンス

| ID | 要件 | 基準 |
|---|---|---|
| NF-001 | 画面遷移速度 | 200ms以内（フェードトランジション） |
| NF-002 | スケジュールスクロール | 指追従でシームレス（フレーム落ちなし） |
| NF-003 | DB問い合わせ | N+1クエリ禁止。一括取得方式 |
| NF-004 | CustomPainter | Paintオブジェクトをキャッシュ |

### 5.2 セキュリティ

| ID | 要件 | 基準 |
|---|---|---|
| NF-005 | XSS対策 | ユーザー入力をHTML/CSSに埋め込む箇所はサニタイズ |
| NF-006 | SQLインジェクション対策 | Drift ORMパラメータ化クエリのみ使用 |
| NF-007 | 入力バリデーション | 全入力フィールドにバリデーション設定 |
| NF-008 | インポート検証 | JSONインポート時にサイズ上限（10MB）・型チェック |
| NF-009 | 機密情報管理 | APIキー・シークレットのハードコード禁止 |

### 5.3 保守性

| ID | 要件 | 基準 |
|---|---|---|
| NF-010 | ラベル外出し | 全UIテキストを `app_labels.dart` で一元管理 |
| NF-011 | テストカバレッジ | 全ソースコードにテスト対応 |
| NF-012 | 静的解析 | `flutter analyze` エラー0件 |
| NF-013 | CI/CD | GitHub Actions で自動テスト・デプロイ |

## 6. 制約条件

- Web版はGitHub Pages（静的ホスティング）で提供するため、サーバーサイド処理は不可
- データベースはクライアントサイドSQLite（Drift ORM + WASM）
- Firebase は認証とFirestore同期のみ使用（サーバーキー禁止）
- GitHub Actions の無料枠で運用するため、テスト/デプロイはmain push時のみ

## 7. 用語定義

| 用語 | 定義 |
|---|---|
| 夢 | ユーザーの長期的な願望・ビジョン |
| 目標 | 夢を実現するための具体的なステップ（What/When/How） |
| タスク | 目標を達成するための日々の行動項目 |
| スケジュール | タスクのタイムライン表示（旧: ガントチャート） |
| 活動ログ | タスクに対する活動時間の記録 |
| 星座 | 活動の積み重ねで完成するゲーミフィケーション要素 |
| 受信ボックス | リマインダー・実績・開発者通知の統合受信箱 |
